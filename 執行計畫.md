# 芭拉咘咘麵包車形象版 — 執行計畫

## 專案概述
芭拉咘咘麵包車的**形象官網**，靈感來自 Purebread Bakery 的簡約乾淨風格，但保有原本 V2 的後台管理與訂單流程。

## 資料夾結構
```
芭拉咘咘麵包車形象版/
├── index.html          ← 主程式（所有頁面 + CSS + JS 合一）
├── 執行計畫.md         ← 本檔案
└── 網址.txt            ← 部署後產生的網址
```

## 頁面規劃（單頁 SPA，分頁切換）

| 分頁 | 說明 |
|------|------|
| 🏠 **首頁** | 品牌形象輪播 + 精選商品 + 店家簡介 |
| 🛵 **選擇麵包車** | 列出所有車輛，點選後進入該車的商品頁 |
| 🍞 **商品** | 該車的分類 tab + 品項網格（含圖片、名稱、價格、+按鈕） |
| 🛒 **購物車** | 增減數量、小計、總計、結帳（限單一車輛） |
| 📋 **訂單** | 待取貨 / 已完成狀態，管理員可結單扣庫存 |
| ℹ️ **關於** | 品牌故事 + Google 地圖 + 地址/電話 |
| ⚙️ **管理** | 車輛管理 → 選車 → 品項/庫存/主題色/店家資訊 |

## 多車架構
- 後台可新增/編輯/刪除車輛（車名可自訂）
- 每台車有**獨立的分類、品項、庫存、顯示開關**
- 前台：消費者先選車 → 才看到該車的商品
- 購物車限裝**同一台車**的商品，換車提示清空

## 品項結構
```
分類（e.g. 歐式麵包）
  └── 品項（e.g. 酸種麵包）
        ├── 單位：個
        ├── 顯示於官網：ON / OFF
        ├── 圖片網址（選填，後台預設隱藏，點才展開）
        └── 價位規格（可多個）
              ├── "單個" $150 → 庫存扣 1
              └── "三個" $400 → 庫存扣 3
```

## 庫存管理
- 沿用 V2：「數量型標籤」決定扣存量（"3" → 扣 3）
- 訂單送出 → 管理員結單 → 自動扣庫存
- Firestore 即時同步庫存

## 訂單流程
1. 消費者選車 → 加商品到購物車 → 結帳
2. 訂單存入 Firestore（標記車輛名稱）
3. 管理員在後台看到訂單 → 結單 → 庫存扣除
4. 消費者可在訂單頁查看「待取貨」狀態

## 設計風格
- **預設色系**：粉紅主色（primary `#E8A0B4`、bg `#FFF5F7`）
- **可切換**：5 套主題（出爐暖棕、鄉村田園、文青簡約、甜蜜粉嫩、復古歐風）+ 自訂色盤
- **版面**：簡約乾淨、留白多、圓角卡片、柔和陰影
- **RWD**：手機優先，平板/桌面自動適配

## 技術選型
- 單頁 HTML（一個 `index.html`）
- CSS Variables 實現主題切換
- Firebase Firestore（儲存車輛、品項、訂單、庫存、設定）
- localStorage（快取購物車資料）
- Google Maps Embed（關於頁）

## 部署方式
- 透過 `node _tools\upload.js 芭拉咘咘麵包車形象版` 上傳到 Firebase
- 網址格式：`https://opencode-sk.web.app/projects/芭拉咘咘麵包車形象版/`
